Documentação

Este projeto está organizado de forma que a sua estrutura de pastas segue uma abordagem simplificada, em que a aplicação se divide em partes específicas, por meio dos Controllers, Services, Directives e Views.

Para o gerenciamento de dependências foi utilizado o bower, dessa forma, ao efetuar o clone e/ou realizar o download do projeto, é preciso executar a instrução “bower install” para que as dependências sejam carregadas e o projeto seja executado corretamente.

Os dados iniciais estão vindo de uma API "mockada", que ao ser carregado no “config.js” armazena seus valores no localStorage do browser, a fim de que as informações tratadas sejam mantidas enquanto o usuário estiver utilizando a aplicação, até que seja limpado os dados do navegador.

Utilizou-se o HTTP-SERVER como servidor HTTP para a execução do projeto, portanto, recomenda-se a sua utilização para a eficácia na utilização desse projeto
